The best threshold isn't statistical - it's economic.

A ripe-melon detector doesn't have one right confidence threshold. It has a family of them, and picking one is an engineering decision. The right choice falls out of what each mistake costs the harvest: a missed ripe melon is lost revenue; a false alarm is only a wasted pick cycle. Here is that decision, derived and made draggable.

How the operating point is chosen

The model gives every candidate a confidence score. A threshold τ\tauτ decides which become picks. We follow Module 4 Sec. 7 exactly: define the metrics, expose the trade, then let the mission choose the point.

Precision and recall count the two ways the detector can be right or wrong.
Precision=TPTP+FPRecall=TPTP+FN\text{Precision} = \frac{TP}{TP + FP} \qquad \text{Recall} = \frac{TP}{TP + FN}Precision=TP+FPTP​Recall=TP+FNTP​

After matching detections to ground truth, count true positives TPTPTP, false positives FPFPFP, and false negatives FNFNFN. Precision asks how many picks were real fruit; recall asks how many real fruit we found. Sweeping the threshold τ\tauτ traces the precision-recall (PR) curve.

Consequence

One model presents a whole family of behaviors - the PR curve - not a single fixed performance.

The two metrics trade against each other - so there is no best threshold in the abstract.

Lowering τ\tauτ admits more detections: recall rises, precision falls. There is no single best τ\tauτ in the abstract - only a best τ\tauτ for a stated objective. And the objective on this machine is economic, not statistical.

Consequence

Choosing τ\tauτ is an engineering decision, and it needs an objective drawn from the mission - not from a convention.

Price each outcome by its mission consequence, then maximize expected value.

A true positive earns the harvested-fruit value vHv_HvH​; a false negative forfeits it (lost yield); a false positive costs a wasted pick cycle cFc_FcF​, spent against the cycle-time budget (CEC-01). Because a missed ripe melon (vHv_HvH​) is worth far more than a wasted cycle (cFc_FcF​) - and wasted cycles are bounded by the cycle-time margin, not catastrophic - the optimum biases toward high recall.

Consequence

This is the quantitative reason a harvester is tuned for recall, and the ratified target follows: recall >= 90% at the operating point (Doc B Sec. 5.4).

Engineering Insight EI-04 - the metric that maps to the mission

Optimize the number that actually matters

EI-04 - Optimize the Metric That Maps to the Mission

Choose and tune your performance metric from the real-world consequence of each error - not from what is conventional, symmetric, or easy to report. The two errors here are not equal: a missed ripe melon is lost revenue, a false alarm is only a wasted cycle. Tuning to a symmetric score like accuracy or F1 silently optimizes the wrong objective. Write down what each error costs the mission, pick the operating point that maximizes mission value, and only then reach for a bigger model if that point still isn't good enough. The trap this avoids is leaderboard engineering - improving a number that is easy to measure while the thing that matters gets worse.
